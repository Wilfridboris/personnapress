## Visual Identity

- **D-01** Aesthetic: "Paper Style" — brutalist, academic, Notion-editorial. Off-white paper ground, deep black ink, sharp corners, flat depth. No gradients. No shadows except optional hard brutalist offset.
- **D-02** Color: Background #F9F9F6, Ink #111111, Graphite #555555, Border #E5E5E5, Highlighter #FFF1B8, Danger #8B0000, Success #2E4F2E. Source: design.prd.md §2.
- **D-03** Typography: Playfair Display for headings (H1/H2), Inter for body/UI elements, JetBrains Mono for Brain Dump input and system logs. Source: design.prd.md §3. [ASSUMPTION: Google Fonts delivery for all three.]
- **D-04** Corners: rounded-none for cards, inputs, modals (brutalist). [ASSUMPTION: 2px rounding only on status badges/pills for legibility.]
- **D-05** Shadows: Hard offset shadow (4px 4px 0px #111111) on interactive cards and primary buttons only. No soft/blur shadows anywhere.
- **D-06** Buttons: Primary = black fill, white text, hard offset shadow, inverts on hover. Secondary = white fill, 1px black border, no shadow.
- **D-07** Inputs: Transparent fill, bottom-border-only (1px #111111), monospace font for Brain Dump. No focus ring; bottom border thickens to 2px on focus.
- **D-08** Loading state: Typewriter animation for AI generation feedback. Source: PRD FR-15, design.prd.md §4.
- **D-09** Tone tokens: Danger (#8B0000) for destructive actions and Reject status. Success (#2E4F2E) for Approve and Published status.

## Information Architecture

- **IA-01** Primary navigation: Left sidebar, persistent on desktop (≥1024px). [ASSUMPTION: Sidebar collapses to icon-only on 768–1023px, drawer on <768px.]
- **IA-02** Client switcher: At top of sidebar above main nav links. Active client name always visible.
- **IA-03** Surfaces defined: Dashboard, Brain Dump (page overlay), Approval Gate (full page), Client Management, Brand Voice Profile, Platform Connections, Content Calendar, Account/Subscription, Onboarding.
- **IA-04** Dashboard is the landing surface post-login for all returning users.
- **IA-05** Onboarding is a separate flow (not the Dashboard) triggered on first login. Three steps: Create Client → Brand Ingestion → First Brain Dump.

## UX Blockers Resolved (PRD §13)

- **UX-B1 Voice questionnaire (PRD Q1):** [ASSUMPTION: Multi-step form. Step 1 — Tone sliders (Formal/Casual, Professional/Friendly, Concise/Elaborate) each 1–5 scale. Step 2 — Paste up to 3 sample text excerpts the user considers representative of their voice. Step 3 — Optional: paste URLs of writers/publications whose style they admire. No more than 3 screens. Submit triggers FR-10 voice extraction with questionnaire data as the source corpus.]
- **UX-B2 Webflow Collection mapping (PRD Q4):** [ASSUMPTION: After Webflow OAuth connection, the system fetches the user's Webflow Collections list via Webflow CMS API and presents a dropdown. User selects the target Collection. Selection is saved on the Platform Connection record. No manual ID entry required. If the API call fails, fallback to a text input for the Collection ID with documentation link.]

## Interaction Decisions

- **INT-01** Approval Gate is a full-page experience, not a modal — content density (blog preview + 2 social posts + image) requires full viewport.
- **INT-02** Brain Dump entry point: primary CTA on Dashboard ("New Campaign" button). Opens a focused overlay (full-page on desktop, native full-screen on mobile).
- **INT-03** Campaign list: newest first. Status badge color-coded: pending_approval = Highlighter fill, approved = Graphite fill, published = Success fill, rejected = no fill + strikethrough, failed = Danger fill.
- **INT-04** Inline blog editing in Approval Gate: WYSIWYG (PRD FR-19 A-18). Social post editing: plain text with live character count.
- **INT-05** Scheduled publish: datetime picker inline in Approval Gate after approval. [ASSUMPTION: Timezone display uses account-level timezone setting (PRD A-21) shown alongside the picker.]
- **INT-06** Image regeneration: "Regenerate image" button shows remaining attempts (e.g., "Regenerate (2 left)"). Disabled when cap reached.
- **INT-07** Platform connection flows: OAuth platforms (X, LinkedIn) open a popup window. Credential platforms (WordPress, Webflow) use an inline form within a side panel.
- **INT-08** Trial upgrade banner: Sticky top-of-page banner post-trial expiration. Non-dismissible. Contains plan name, expiry date, and "Subscribe" CTA.
- **INT-09** Voice fidelity badge: When score fails threshold, shown in Approval Gate header as "Voice match: N/10 — review tone" in Danger color. Advisory only — does not block Approve.
- **INT-10** Destructive actions (Delete Client, Reject Campaign): require a confirmation dialog naming the specific item and downstream consequences.
